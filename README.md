# 🚖 Taxi App

Educational BA pet project for analyzing and modeling an online taxi ordering platform.

The system allows customers to order rides, drivers to accept trips, and administrators to manage the platform, payments, and ride activity.

---

# 📌 Problem

- Difficult ride management
- Slow driver assignment
- Lack of real-time ride tracking
- Inefficient payment processing
- Poor transportation workflow organization

---

# 🎯 Project Goal

Design and analyze a taxi ordering system that supports:
- fast ride requests
- automatic driver matching
- secure payment processing
- ride tracking
- driver rating system

---

# ⚙️ Features

✅ User registration & authentication  
✅ Taxi ride ordering  
✅ Nearby driver search  
✅ Automatic ride price calculation  
✅ Ride status tracking  
✅ Online & cash payments  
✅ Driver rating system  
✅ Ride history  
✅ Admin dashboard  
✅ API structure modeling  

---

# 👥 System Actors

| Actor | Role |
|---|---|
| Customer | Creates taxi orders and pays for rides |
| Driver | Accepts and completes rides |
| Administrator | Manages users, tariffs, and system activity |
| Support Team | Handles customer issues and support requests |
| Payment Provider | Processes payment transactions |
