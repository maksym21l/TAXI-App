# API Документація — Taxi App

> Цей документ описує ключові REST API endpoints платформи Taxi App.  
> API використовується мобільним додатком пасажира, мобільним додатком водія та адмін-панеллю.

---

## Base URL

```
https://api.taxiapp.com/v1
```

---

## Автентифікація

Всі endpoints (крім реєстрації та логіну) вимагають Bearer token у заголовку запиту.

```
Authorization: Bearer <token>
```

Токени видаються після успішного логіну та дійсні протягом 24 годин.  
Якщо токен відсутній або прострочений — сервер повертає `401 Unauthorized`.

---

## Огляд endpoints

| Метод | Endpoint | Опис | Авторизація |
|-------|----------|------|-------------|
| POST | /auth/register | Реєстрація нового користувача (пасажир або водій) | Ні |
| POST | /auth/login | Вхід в систему та отримання токена | Ні |
| POST | /auth/logout | Завершення поточної сесії | Так |
| POST | /rides/request | Створення нового запиту на поїздку | Так |
| GET | /rides/{id} | Отримання деталей поїздки за ID | Так |
| PATCH | /rides/{id}/cancel | Скасування активної поїздки | Так |
| PATCH | /rides/{id}/complete | Позначення поїздки як завершеної (тільки водій) | Так |
| GET | /drivers/nearby | Отримання списку найближчих доступних водіїв | Так |
| POST | /payments | Обробка оплати за поїздку | Так |
| POST | /ratings | Відправлення рейтингу після завершення поїздки | Так |
| GET | /rides/history | Отримання історії поїздок поточного користувача | Так |

## Деталі endpoints

---

### POST /auth/register

Реєстрація нового акаунту пасажира або водія.

**Тіло запиту:**
```json
{
  "role": "passenger",
  "first_name": "Анна",
  "last_name": "Коваленко",
  "phone": "+380991234567",
  "password": "securepassword123"
}
```

> Для водіїв додатково потрібні поля: `vehicle_brand`, `vehicle_model`, `plate_number`, `vehicle_category`.

**Відповідь `201 Created`:**
```json
{
  "id": "usr_001",
  "role": "passenger",
  "phone": "+380991234567",
  "status": "active"
}
```

**Можливі помилки:**

| Статус | Причина |
|--------|---------|
| 400 | Відсутні обов'язкові поля |
| 409 | Номер телефону вже зареєстровано |

---

### POST /auth/login

Вхід за номером телефону та паролем.

**Тіло запиту:**
```json
{
  "phone": "+380991234567",
  "password": "securepassword123"
}
```

**Відповідь `200 OK`:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_at": "2025-06-01T12:00:00Z",
  "user_id": "usr_001",
  "role": "passenger"
}
```

**Можливі помилки:**

| Статус | Причина |
|--------|---------|
| 400 | Відсутній телефон або пароль |
| 401 | Невірні облікові дані |

---

### POST /rides/request

Створення нового запиту на поїздку. Система автоматично шукає найближчого доступного водія.

**Тіло запиту:**
```json
{
  "pickup_location": "вул. Хрещатик, 1, Київ",
  "destination_location": "вул. Велика Васильківська, 100, Київ",
  "vehicle_category": "comfort",
  "payment_method": "card"
}
```

**Відповідь `201 Created`:**
```json
{
  "ride_id": "ride_042",
  "status": "searching",
  "estimated_price": 95.00,
  "currency": "UAH",
  "vehicle_category": "comfort",
  "payment_method": "card"
}
```

**Можливі помилки:**

| Статус | Причина |
|--------|---------|
| 400 | Недійсна або нерозпізнана адреса |
| 404 | Немає доступних водіїв поблизу |
| 402 | Карткову оплату відхилено |

---

### GET /rides/{id}

Отримання поточного статусу та деталей конкретної поїздки.

**Відповідь `200 OK`:**
```json
{
  "ride_id": "ride_042",
  "status": "in_progress",
  "pickup_location": "вул. Хрещатик, 1, Київ",
  "destination_location": "вул. Велика Васильківська, 100, Київ",
  "driver": {
    "name": "Олексій М.",
    "rating": 4.8,
    "vehicle": "Toyota Camry • АА1234ВВ"
  },
  "price": 95.00,
  "payment_method": "card"
}
```

**Значення статусів поїздки:**

| Статус | Значення |
|--------|---------|
| `searching` | Система шукає водія |
| `accepted` | Водій прийняв замовлення |
| `in_progress` | Поїздка активна |
| `completed` | Поїздка успішно завершена |
| `cancelled` | Поїздку скасовано |

---

### POST /ratings

Відправлення рейтингу після завершення поїздки. Пасажири та водії можуть оцінювати один одного.

**Тіло запиту:**
```json
{
  "ride_id": "ride_042",
  "score": 5,
  "comment": "Дуже приємна поїздка, водій пунктуальний"
}
```

**Відповідь `201 Created`:**
```json
{
  "rating_id": "rat_011",
  "ride_id": "ride_042",
  "score": 5,
  "saved": true
}
```

**Можливі помилки:**

| Статус | Причина |
|--------|---------|
| 400 | Оцінка поза діапазоном (має бути від 1 до 5) |
| 403 | Поїздка не завершена — рейтинг недозволений |
| 409 | Рейтинг для цієї поїздки вже надіслано |

---

## Загальний формат помилок

Всі помилки мають однакову структуру відповіді:

```json
{
  "error": {
    "code": "RIDE_NOT_FOUND",
    "message": "Поїздки з вказаним ID не існує.",
    "status": 404
  }
}
```

---

## Примітки

- Всі часові мітки у форматі **ISO 8601** (`2025-05-22T10:00:00Z`)
- Всі грошові значення в **UAH**, якщо не вказано інше
- Акаунти водіїв потребують верифікації адміністратором перед початком роботи (див. FR-02, BR-08)
- Рейтинги можна надсилати лише для поїздок зі статусом `completed` (див. BR-07)
